# Краткое описание
Создала с нуля дашборд в Yandex DataLens для PR-отдела, который позволяет отслеживать публикации в медиаполе и выполнение плана по активности.

Все данные — из разных источников — объединены в одном окне, что упростило аналитику и ускорило принятие решений.

# Контекст и цель
PR-отдел ежедневно работает с большим объёмом информации: планируемые публикации, фактически вышедшие материалы, охваты, медиаиндексы и т.д.

До внедрения дашборда всё велось вручную — в Google Sheets и выгрузках из Медиалогии.

Цель — автоматизировать сбор данных и визуализировать ключевые метрики, чтобы:
    - видеть фактическое выполнение PR-плана в реальном времени;
    - анализировать охваты и медиапоказатели;
    - принимать решения на основе актуальных данных.

# Инструменты и технологии
Python, Google Sheets, API, Git, PostgreSQL, SQL, Yandex DataLens

# Этапы работы

## 1. Анализ требований:
Провела обсуждения с PR-отделом и уточнила:
    - цели дашборда и потребности пользователей
    - ключевые показатели (кол-во публикаций, охват, медиаиндекс и т.д.)
    - источники данных и периодичность обновления

## 2. Разработка архитектурного решения
Собрала архитектуру данных и ETL-пайплайна, объединив Google Sheets, Медиалогию и внутреннюю БД.
![Архитектурное решение](https://github.com/Olysechka02/Portfolio/blob/main/Материалы/Архитектурное%20решение%20для%20дашбордов%20PR.png)

## 3. Реализация ETL-процесса
Построила полный ETL-пайплайн:
    1. Из Google Sheets берутся плановые и фактические публикации.
    2. Из Медиалогии настраивается ежедневная рассылка отчётов.
    3. Через API Gmail автоматически парсятся письма и выгрузки из Медиалогии.
    4. Все данные записываются в PostgreSQL.
    5. На основе этих таблиц создаётся витрина данных для визуализации.

## 4. Построение дашборда
Создала визуализации в DataLens:
    - выполнение плана,
    - динамика публикаций,
    - география и охваты СМИ,
    - распределение по типам СМИ
